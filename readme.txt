Этот проект был выполнен для того, чтобы разобраться с утройством и функционалом хранилища данных,
возможными методами моделирования данных, методами поддержания историчности записей и etl и elt процессами между слоями хранилища.

Хранилище содержит в себе информацию о фильмах, находящихся в топ-5000 рейтингах по пользовательским оценкам на ресурсах IMDB и Metacritic. 
При обновлении рейтингов и повторном парсинге данных с ресурсов, раскладка в слои хранилища выполняется с закрытием дат неактуальных записей в случае 
изменения рейтинга или любых других изменениях данных о фильме и созданием новой, актуальной записи. В случае же отсутсвия изменений запись остается открытой.
Таким образом, данное хранилище фильмов делает возможным отслеживание колебания рейтинга, сборов и других данных о фильме в течении длительного периода.

В папках 'core' и 'mart' находятся скрипты расчета таблиц ядра и витринного слоев хранилища соответственно.
Воссоздать работу хранилища можно таким образом:
	1. Создать пустую базу данных, восстановить ее из DUMP файла (в таком случае хранилище уже будет содержать все наполненные таблицы/
		/создать пустую базу данных, выполнить 'ddl.py' указав релевантное имя пользователя, пароль и название бд.
	2. Выпонить парсинг данных рейтинга IMDB выполнением скрипта из 'imdb parser.py' (запускать строго без VPN), также указав имя пользователя, пароль и название бд.
	3. Выпонить парсинг данных рейтинга METACRITIC выполнением скрипта из 'metacritic parser.py' (запускать с VPN, в РФ сайт не работает), также указав имя пользователя, пароль и название бд.
	4. После завершения парсинга выполнить скрипт 'etl_layer_transfer.py' для осуществления перекладки данных в верхние слои хранилища. Здесь также нужно указать пароль, имя пользователя и название бд.
